# Proposed: Specific Retrieval Hooks

## Classification

ADD

## Durable Idea

Retrieval hooks should include specific task terms, filenames, decisions, error strings, or identifiers because vague hooks can cause memory lookup failure.

## Rationale

The inbox supplied a source-stamped candidate describing a failed retrieval caused by a vague memory index hook. The active retrieval fixture already identifies this as a relevant failure mode, so this candidate is suitable for human review rather than automatic active promotion.

## Risk

This may duplicate existing index maintenance guidance. A reviewer should decide whether it adds durable value or should remain only as an episodic log entry.

## Approval

Human approval is required before this proposed memory can be promoted, indexed, or used to supersede any active memory.
